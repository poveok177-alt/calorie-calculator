## 🚀 Быстрый старт (1 минута)

### Локально (Development)

```bash

cp -r /mnt/user-data/outputs/* ~/my-calorie-tracker/

# 2. Создайте виртуальное окружение
cd ~/my-calorie-tracker
python3 -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate

# 3. Установите зависимости
pip install -r requirements.txt

# 4. Запустите приложение
python app.py

# 5. Откройте браузер
# http://localhost:5000
```


## 🔧 Как всё работает (технически)

### User → favorites (JSON)
```python
# В БД хранится как:
favorites = '[1, 5, 23, 45]'  # массив ID продуктов

# При добавлении в избранное:
favs = json.loads(current_user.favorites or '[]')
favs.append(food_id)
current_user.favorites = json.dumps(favs)
db.session.commit()
```

### meal_type → FoodEntry
```python
# При добавлении продукта:
entry = FoodEntry(
    user_id=current_user.id,
    food_id=food.id,
    grams=200,  # пользователь ввёл 200г
    meal_type='breakfast',  # выбрал Завтрак
    calories=round(food.calories * 200/100, 1),  # пересчитал на 200г
    ...
)

# На главной группируем:
meals = {
    'breakfast': [entry1, entry2, ...],
    'lunch': [entry3, entry4, ...],
    'dinner': [...],
    'snack': [...]
}
```

### API /api/recent
```python
# Возвращает последние 10 добавленных (по created_at)
# Без дубликатов по food_id (берём первое вхождение)
entries = FoodEntry.query.filter_by(user_id=current_user.id)\
    .order_by(FoodEntry.created_at.desc()).limit(10).all()
# Берём food для каждого entry, убираем дубли
```

## 🐛 Troubleshooting

### Q: После запуска ошибка "ALTER TABLE"
**A:** Это нормально! Приложение автоматически добавит нужные колонки.

### Q: Добавил продукт, но он не видно в приёме пищи
**A:** Проверьте что выбрали правильный приём пищи в модали. Старые записи без meal_type не покажутся.

### Q: Избранные не сохраняются
**A:** Проверьте консоль браузера (F12). Если ошибка 401 — нужно заново войти.

### Q: Поиск работает странно
**A:** Поиск ищет по названиям на выбранном языке. Меняйте язык если не находит.

### Q: Модаль не открывается
**A:** Возможно вы не авторизованы. Кликните на продукт после входа.

## 🎓 Для разработчиков

### Как добавить ещё витаминов/спортпита

Отредактируйте `food_data.py`:

```python
{
    "name_ru": "Витамин E (капсула)",
    "name_en": "Vitamin E (capsule)",
    "name_uk": "Вітамін E (капсула)",
    "name_kk": "E Витамины (капсула)",
    "calories": 5,
    "protein": 0,
    "fat": 0.3,
    "carbs": 0.2,
    "category": "supplements"  # или "sports_nutrition"
}
```

Затем:
```bash
# Удалите старую БД (или обновите вручную)
rm calories.db
python app.py  # Пересоздаст БД с новыми продуктами
```

### Как добавить новый язык

В `app.py` в `TRANSLATIONS`:

```python
'es': {  # Испанский
    'app_name': 'CaloriMint',
    'tagline': 'Tu cuidado corporal comienza aquí',
    # ... и так для каждого ключа
}
```

Затем в `CATEGORY_KEYS` добавьте язык в словарь `name_col`.
