## 🧪 Как тестировать

### 1. Разбивка по приёмам пищи
```
1. Войти в приложение
2. На главной странице должны быть 4 блока: Завтрак, Обед, Ужин, Перекус
3. Добавить продукт (поиск)
4. В модали выбрать приём пищи
5. Продукт должен появиться в соответствующем блоке
6. Калории в блоке должны пересчитаться
```

### 2. Недавние и избранные
```
1. Добавить 3-4 разных продукта
2. Кликнуть на вкладку "⏰ Недавние" — должны показаться последние продукты
3. Кликнуть ⭐ рядом с какимм-то продуктом — звездочка пожелтеет
4. Перейти на вкладку "⭐ Избранные" — продукт должен быть там
5. Обновить страницу — избранные должны остаться (сохранены в БД)
```

### 3. Граммовка
```
1. Добавить продукт из поиска
2. В модали изменить вес с 100 на 200
3. Калории должны удвоиться, БЖУ пересчитаться
4. Изменить обратно на 50
5. Значения должны быть в половину меньше от оригинала
6. Нажать "Добавить" — продукт добавится с правильным весом и калориями
```

### 4. Новые продукты
```
1. В поиске вбить "протеин" или "витамин"
2. Должны появиться новые продукты со спортпита и витаминов
3. Добавить их (проверить что граммовка работает)
4. Посмотреть в категориях — должны быть новые иконки 💊 и 💪
```

## ⚠️ Важно знать

### Избранные — это JSON в БД
Хранятся в виде: `[1, 5, 23, 45]` (массив ID продуктов)
- При каждом добавлении в избранное — JSON обновляется
- Если JSON повреждён — функция перехватит исключение и вернёт пустой массив

### Meal type теперь обязателен
- Старые записи (если есть) могут иметь пустой meal_type
- На главной они не будут видны (требуется meal_type)
- Новые записи всегда имеют meal_type

### История (30 дней) не обновлена
- Эта функция из Приоритета 2
- Страница /history работает как раньше (показывает все)
- График калорий — в планах на v2.1

## 📞 Возможные проблемы и решения

### Модаль не открывается
**Причина**: `isAuth` = false
**Решение**: Убедитесь что вы вошли в приложение

### Недавние не загружаются
**Причина**: Ещё не добавили ни один продукт
**Решение**: Добавьте хотя бы один продукт через поиск

### Избранные не сохраняются
**Причина**: Возможно, ошибка в JSON
**Решение**: Проверьте в консоли браузера (F12 → Console)

### Граммовка считает неправильно
**Причина**: Исходный продукт имеет неправильные значения
**Решение**: Проверьте food_data.py
