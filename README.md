# CaloriMint - Трекер калорий 🌿

Веб-приложение для отслеживания калорийности и БЖУ с красивым минималистичным дизайном.

## ✨ Новые функции (Приоритет 1)

### 🔴 Разбивка по приёмам пищи
- Все продукты за день разделены на **4 секции**: Завтрак, Обед, Ужин, Перекус 🌅 ☀️ 🌙 🍿
- Каждая секция показывает **сумму калорий** внутри
- При добавлении продукта пользователь выбирает приём пищи в модальном окне
- **Цветная кодировка** продуктов по типам (оранжевый, зелёный, синий, розовый)

### 🔴 Быстрое повторное добавление
- **Вкладка "Недавние"** — последние 10 добавленных продуктов
- **Вкладка "Избранные"** — помеченные ⭐ продукты
- Значок **⭐** на каждом продукте для быстрого сохранения в избранное
- Избранные хранятся в профиле пользователя (JSON в БД)

### 🔴 Граммовка при добавлении
- При клике на продукт открывается **модальное окно**
- Поле для ввода **веса в граммах** (по умолчанию 100г)
- **Автоматический пересчёт** КБЖУ пропорционально весу
- Выбор приёма пищи в модальном окне (Завтрак/Обед/Ужин/Перекус)
- Показ **итоговых значений** до подтверждения

## 🔧 Установка и запуск

### Локально
```bash
# 1. Клонируем репозиторий
git clone https://github.com/ваш-ник/calorie-calculator.git
cd calorie-calculator

# 2. Создаём виртуальное окружение
python3 -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate

# 3. Устанавливаем зависимости
pip install -r requirements.txt

# 4. Запускаем приложение
python app.py
```

Приложение будет доступно по адресу: **http://localhost:5000**

### На Railway/Heroku
```bash

# 2. Подключите PostgreSQL
# 3. Установите переменные окружения:
DATABASE_URL=postgresql://...
FLASK_ENV=production

# 4. Задеплойте!
git push heroku main  # или Railway
```

## 📊 Структура БД

### Таблица `user`
```python
- id (Integer, PK)
- username, email, password_hash
- language (ru/en/uk/kk)
- is_premium (Boolean)
- current_weight, goal_weight, height
- daily_calorie_goal, protein_goal, fat_goal, carbs_goal
- water_goal, age, gender, activity
- favorites (JSON: [food_id, ...])  # НОВОЕ
- trial_used, trial_ends
```

### Таблица `food`
```python
- id (Integer, PK)
- name_ru, name_en, name_uk, name_kk (multilingua)
- calories, protein, fat, carbs (на 100г)
- category (fruits, vegetables, meat, dairy, grains, 
           nuts, fish, sweets, drinks, supplements, 
           sports_nutrition, other)  # НОВОЕ: supplements, sports_nutrition
```

### Таблица `food_entry`
```python
- id (Integer, PK)
- user_id, food_id (FK)
- food_name, grams
- calories, protein, fat, carbs (уже пересчитано)
- date, meal_type (breakfast/lunch/dinner/snack)  # ОБНОВЛЕНО: meal_type
- created_at
```

## 🔐 API Эндпоинты

### Аутентификация
- `POST /register` — Регистрация
- `POST /login` — Вход
- `GET /logout` — Выход
- `GET /set-language/<lang>` — Смена языка

### Основные страницы
- `GET /` — Главная (сегодня + приёмы пищи)
- `GET /goals` — Мои цели
- `GET /history` — История за месяц
- `GET /categories` — Категории продуктов
- `GET /premium` — Информация о премиуме

### API
- `GET /api/search?q=слово` — Поиск продуктов
- `GET /api/recent` — Последние 10 продуктов **[НОВОЕ]**
- `GET /api/favorites` — Избранные продукты **[НОВОЕ]**
- `POST /api/favorites` — Добавить в избранное **[НОВОЕ]**
- `DELETE /api/favorites` — Удалить из избранного **[НОВОЕ]**
- `POST /api/add-entry` — Добавить продукт (с meal_type)
- `DELETE /api/remove-entry/<id>` — Удалить запись
- `DELETE /api/clear-day` — Очистить день
- `GET /api/water` — Информация о воде
- `POST /api/water` — Обновить воду
- `GET /api/today-summary` — Сводка за сегодня

## 📝 Примеры использования

### Добавить продукт
1. Кликнуть на поиск или вкладку "Недавние"
2. Выбрать продукт из списка
3. В модальном окне:
   - Ввести вес в граммах (можно оставить 100)
   - Выбрать приём пищи
   - Посмотреть пересчитанные КБЖУ
4. Кликнуть "✓ Добавить"

### Добавить в избранное
1. При поиске кликнуть ⭐ рядом с названием продукта
2. Звёздочка покрасится жёлтым
3. Продукт появится во вкладке "⭐ Избранные"

### Посмотреть приёмы пищи
- На главной странице видны 4 секции
- Каждая показывает калории и список продуктов
- Можно удалить продукт нажав ✕

## 🛠 Troubleshooting

### ModuleNotFoundError: No module named 'yookassa'
Это нормально! YooKassa опциональна. Если не используете платежи — игнорируйте ошибку.

### Ошибка миграции БД
Если получили ошибку с ALTER TABLE, выполните:
```python
python3
from app import db, app
with app.app_context():
    db.create_all()
```

### Разделы не видны на мобильном
Адаптивный дизайн работает. Проверьте viewport:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
